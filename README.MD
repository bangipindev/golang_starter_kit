# рҹ§  Architecture Overview
```
	Project ini menggunakan Clean Architecture:

	HTTP (Fiber)
	вҶ“
	Handler (Delivery Layer)
	вҶ“
	Usecase (Business Logic)
	вҶ“
	Domain (Interface + Entity)
	вҶ“
	Infrastructure (DB, Repository, Migration)
```


## рҹ“Ң Layer Explanation

### 1пёҸвғЈ delivery/http
Berisi:
- DTO (request/response struct)
- Handler (controller)
- Route definition

Tidak boleh berisi business logic.

---

### 2пёҸвғЈ domain
Berisi:
- Entity (User struct)
- Interface repository

Pure business model.
Tidak boleh import Fiber / MySQL.

---

### 3пёҸвғЈ usecase ( Bisa juga disebut Layer Service)
Berisi:
- Business logic
- Auth logic
- JWT creation
- Password hashing

Tidak tahu HTTP.
Hanya tahu interface repository.

---

### 4пёҸвғЈ infrastructure
Berisi:
- Database connection
- Repository implementation ( Penulisan Query Ada Disini)
- Migration runner

Layer ini implementasi teknis.

---

### 5пёҸвғЈ middleware
JWT middleware untuk proteksi endpoint.

---

### 6пёҸвғЈ pkg/response
Standard API response wrapper agar konsisten.

```

```

# рҹ”җ API Endpoints

## Register

POST `/api/auth/register`

```
	{
		"name": "Ervin",
		"email": "ervin@mail.com",
		"password": "123456"
	}
```


---

## Login

POST `/api/auth/login`

Response:

```
	{
		"success": true,
		"data": {
			"token": "JWT_TOKEN"
		}
	}
```


---

## Profile (Protected)

GET `/api/profile`

Header:


```
	Authorization: Bearer JWT_TOKEN

```

# рҹҸ— Clean Architecture Rules

вң” Domain tidak boleh import Fiber
вң” Usecase tidak boleh import database
вң” Handler tidak boleh akses database langsung
вң” Repository implementasi ada di infrastructure
вң” Semua dependency di inject dari main.go
